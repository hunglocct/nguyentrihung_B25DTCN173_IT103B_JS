Giới Thiệu Dự Án

1.1 Mục đích

Xây dựng hệ thống quản lý kho đơn giản chạy trên trình duyệt nhằm rèn luyện tư duy Lập trình hàm (Functional Programming).

Loại bỏ các cấu trúc lặp for truyền thống, thay thế bằng các phương thức xử lý mảng hiện đại.

Công nghệ lõi

Khai thác tối đa sức mạnh của bộ Higher-Order Functions:

  • filter() • map() • reduce()
  • some() • every() • forEach()

Phạm vi & Giới hạn

  • Môi trường: Browser-based, JavaScript thuần ES6+.
  • Giao tiếp: prompt() để nhận lệnh và alert() để báo cáo.
  • Ràng buộc kỹ thuật: Tuyệt đối không sử dụng Object.
  • Lưu trữ: Mô hình Parallel Arrays (Mảng song song).
  • Database: In-memory (Dữ liệu mất khi refresh trang).

Kiến thức hệ thống

Menu Loop Control

Sử dụng cơ chế Event Loop đơn giản qua vòng lặp while(true).

  • Hiển thị Menu chức năng.
  • Dùng switch-case để phân nhánh logic.
  • Thoát ứng dụng khi người dùng chọn Option 7.

Cấu Trúc Dữ Liệu

Dữ liệu được quản lý qua 3 mảng riêng biệt nhưng có chỉ số (index) gắn kết chặt chẽ:

Chỉ số (i) names[] prices[] stocks[]
0 iPhone 15 Pro 1000 5
1 Samsung S24 Ultra 900 0
... ... ... ...
// QUY TẮC: names.length === prices.length === stocks.length

Danh Sách Use Case

Lọc Hàng

Sản phẩm cao cấp > 500$

Kiểm Định

Kiểm tra tồn kho & giá sàn

Vốn Hóa

Tổng giá trị tài sản kho

Giảm Giá

Chiến dịch ưu đãi -10%

USE CASE 01

LỌC HÀNG CAO CẤP

UC 1: Thuật Toán Filter

Logic xử lý

Dùng filter trên mảng names nhưng điều kiện lọc dựa trên index tương ứng của mảng prices.

let premium = names.filter((name, i) => { return prices[i] > 500; });

Kết quả trả về một mảng mới chứa tên các sản phẩm đắt tiền.

Use Case 2: Kiểm Định

Xác thực dữ liệu

Sử dụng some và every để kiểm tra tính đúng đắn của toàn kho hàng.

// Có hàng nào hết không? let isOut = stocks.some(s => s === 0); // Giá có hợp lệ (>100) hết không? let isVal = prices.every(p => p > 100);

Use Case 3: Vốn Hóa

REDUCE

Aggregator Pattern

Tính toán tài sản

Dùng reduce để tích lũy giá trị nhân giữa đơn giá và số lượng.

let totalValue = prices.reduce((acc, p, i) => { return acc + (p * stocks[i]); }, 0);

Hàm này quét qua toàn bộ kho và trả về duy nhất 1 con số đại diện cho tổng vốn hóa.

Use Case 4: Giảm Giá

Mutate Original Data

Chiến dịch Marketing yêu cầu giảm giá trực tiếp 10% vào dữ liệu gốc.

prices.forEach((price, i) => { prices[i] = price * 0.9; });

* Quan trọng: forEach không trả về mảng mới, nó tác động trực tiếp lên mảng hiện có.

Cảnh báo đồng bộ

Khi cập nhật giá, chỉ số index phải được bảo toàn tuyệt đối để không làm sai lệch dữ liệu với mảng names và stocks.

Use Case 5: Truy Vấn

Map + Filter Chaining

Kỹ thuật gom dữ liệu vào Object ảo để thực hiện tìm kiếm phức tạp:

let search = names.map((n, i) => ({ n, p: prices[i], s: stocks[i] })) .filter(item => item.n.includes(keyword));

Use Case 6: Báo Cáo

Chuyển đổi trạng thái (Mapping)

Biến đổi dữ liệu thô (số lượng) thành thông tin có nghĩa (văn bản):

let report = stocks.map(s => s > 0 ? "Còn hàng" : "Hết hàng" );

Dễ dàng hiển thị cho người dùng cuối mà không cần xử lý logic phức tạp tại tầng View.

Xử Lý Ngoại Lệ

Input Validation

Kiểm tra isNaN() khi người dùng nhập giá hoặc số lượng qua prompt.

Empty State

Xử lý trường hợp kho hàng rỗng để tránh lỗi khi dùng reduce() hoặc map().
